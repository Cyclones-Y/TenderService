# 招标信息管理平台 - 前端开发需求文档

## 1. 项目概述
本项目旨在构建一个用于浏览、检索、展示及下载招标信息的Web管理平台。目标用户为需要查询和分析招标数据的业务人员。

## 2. 功能模块需求

### 2.1 招标信息列表 (首页/核心页)
**功能描述**：展示系统中采集的所有招标项目信息，支持多条件组合筛选与分页查看。

**UI/UX 需求**：
*   **筛选栏**：
    *   **项目名称**：文本输入框 (支持模糊搜索)
    *   **项目编号**：文本输入框
    *   **所在区县**：下拉选择框 (加载系统中的区县数据)
    *   **项目阶段**：下拉选择框 (如：招标公告、中标候选人、中标结果等)
    *   **发布时间**：日期范围选择器 (开始时间 - 结束时间)
    *   **操作按钮**：查询、重置
*   **数据表格**：
    *   列展示：序号、项目名称 (可点击跳转详情)、项目编号、所在区县、建设单位、项目阶段、发布时间、操作 (详情、下载)。
    *   支持斑马纹显示，行高适中。
    *   列宽支持自适应或手动调整。
*   **分页组件**：
    *   支持切换每页条数 (10, 20, 50, 100)。
    *   显示总条数、当前页码、跳转功能。

### 2.2 招标信息详情页
**功能描述**：展示单个招标项目的完整详细信息。

**UI/UX 需求**：
*   **基础信息卡片**：
    *   项目名称、项目编号、区县、发布时间、公告网站等。
*   **业务信息卡片**：
    *   建设单位、代理机构、项目类型、建设规模、工期等。
*   **金额信息卡片**：
    *   招标控制价、中标价、单方造价等 (金额需格式化，如千分位)。
*   **中标信息卡片** (仅在中标相关阶段显示)：
    *   中标候选人/中标人 (排名1, 2, 3)。
    *   中标下浮率、中标工期等。
*   **原始数据/备注**：
    *   显示备注信息 (如：数据提取失败提示)。
    *   提供“查看原始网页”链接。

### 2.3 数据导出服务
**功能描述**：允许用户将查询结果导出为Excel文件。

**交互需求**：
*   **批量导出**：在列表页勾选多条记录，点击“导出选中”。
*   **全部导出**：点击“导出全部” (根据当前筛选条件导出所有数据，需后端配合流式下载或异步任务)。

### 2.4 仪表盘/数据概览 (建议功能)
**功能描述**：提供数据宏观视角的统计图表。

**图表建议**：
*   **区域分布图**：各区县招标项目数量占比 (饼图/地图)。
*   **阶段分布图**：当前各阶段项目数量统计 (柱状图)。
*   **近期趋势图**：近30天发布招标信息数量趋势 (折线图)。

## 3. 技术要求
*   **框架**：建议使用 Vue 3 + Element Plus 或 React + Ant Design。
*   **响应式**：主要适配 PC 端 (1280px+)，兼顾大屏展示。
*   **交互体验**：
    *   表格加载需有 Loading 状态。
    *   操作成功/失败需有全局 Message 提示。
    *   空数据需展示 Empty 状态图。

## 4. 接口对接说明
*   **列表接口**：`GET /api/tenders` (已实现)
    *   参数：`page_num`, `page_size`, `project_name`, `project_code`, `district`, `project_stage`, `begin_time`, `end_time`
*   **详情接口**：`GET /api/tenders/{tender_id}` (已实现)
    *   返回完整 `TenderModel` 数据结构。
